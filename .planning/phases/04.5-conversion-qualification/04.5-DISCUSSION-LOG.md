## Verrou 3 leads gratuits

| Option | Description | Selected |
|--------|-------------|----------|
| Cumulatif (les 3 premiers, pour toujours) | Un pro qui a vu ses 3 premiers leads en janvier n'en aura plus jamais de gratuits. Le compteur ne se réinitialise pas. Plus simple à implémenter. | ✓ |
| Mensuel (3 leads gratuits chaque mois) | Chaque mois, le pro récupère 3 accès gratuits. Crée une habitude d'usage et incite à revenir. Plus complexe à implémenter (reset mensuel cron). | |

**User's choice:** Cumulatif

---

| Option | Description | Selected |
|--------|-------------|----------|
| Champ booléen is_free sur leads | Les 3 premiers leads d'un pro ont is_free = true. Simple à lire dans maskLead. Résultat déterministe. | |
| Compteur free_leads_used sur professionals | Champ entier incrémenté à chaque accès gratuit. Le seuil (<= 3) est évalué à la requête. Plus flexible si on change le seuil. | ✓ |
| Calculé à la volée (COUNT leads par pro) | Pas de champ supplémentaire. On compte les leads de ce pro à chaque requête. Simple mais moins optimal. | |

**User's choice:** Compteur `free_leads_used` sur `professionals`

---

| Option | Description | Selected |
|--------|-------------|----------|
| Ils restent accessibles, le compteur est ignoré | Premium débloque tout — les règles free_leads_used ne s'appliquent plus tant que l'abonnement est actif. | ✓ |
| Le compteur se réinitialise à 0 quand il revient Basic | Si le pro annule son Premium, il repart de 0 lead gratuit. Incite à rester Premium. | |

**User's choice:** Ignoré en Premium (compteur reprend depuis sa valeur si retour Basic)

---

| Option | Description | Selected |
|--------|-------------|----------|
| Card floutée + bouton Premium (comme avant mais dès le 4ème) | Comportement identique à Phase 4, juste déclenché plus tôt. | ✓ |
| Overlay/modal paywall sur la card | La 4ème card est visible mais bloquée par un overlay. Plus agressif visuellement. | |

**User's choice:** Card floutée + bouton Premium
